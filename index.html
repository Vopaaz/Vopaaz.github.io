代码整洁之道 Notes-1

2019-01-12

寒假阅读的第一本书是《代码整洁之道》，回想起自己以前写代码的时候犯下的一些错误，给自己后面带来的麻烦，书中都有提到，读来确实感觉有很多收获。

在笔记中也主要是记录自己没有注意的内容，将一些可能很重要但是平时已经做到的内容省略。

<!-- more -->

命名

  • 避免误导：编程语言中可能有很多专有类型，例如 List, 如果某个变量不是这种类型，就不要将这个关键字用于命名，例如 accountList, 可以用 accountGroup 等代替

  • 尽量用长名字描述清楚变量的含义，不要使用简短的名字或是缩写

  • 将各种常量都定义为可以搜索的名称，这将便于搜索和后续的修改
    <!–￼0–>

  • 将一整个工程/模块中的同一个概念对应同一个词，例如以下三者不要混用

    • controller
    • driver
    • manager

函数

函数的第一规则是要短小，第二条规则是还要更短小。
函数应该只做一件事。

  • 尽最大可能地拆分函数。

  • if-else 语句和 while 语句中的代码块应当尽量短小，可以拆分另一个函数出来用于专门处理这种情况，尽量避免多重嵌套

  • 对“函数应该只做一件事”的详细解释：做的事情在同一抽象层面上
    <!–￼1–>

  • 函数的整体组织形式应当可以用 “To” 语言来清晰地，自顶向下地描述

    • 描述
      1. To do A, firstly do B, if C, then do D
      2. To do B, firstly do B1, then do B2
      3. To judge C, firstly do E, then judge F
      4. To do D, firstly do D1, then do D2
    • 注意每一行 To 仅仅只能对应 1 个函数，并且这行描述应当是简洁而清晰的。如果过长则说明应当拆分函数
    • 代码：
      <!–￼2–>

函数参数

  • 函数参数地数量应当越少越好，如果超过了 3 个，则必须进行封装，将逻辑上有关联的几个变量封装成一个 class
  • 不要使用“输出参数”，即函数返回值为 void, 而对于其中某一个参数进行转换。这不直观，也不符合正常人的思维方式
    • 当一个函数要对某些实体的状态作出修改时，只能够修改其所属的对象的状态

刚开始学 Python 的时候对于 list 的两个排序分不清楚，看到这里就豁然开朗，而且觉得这是个很好的例子了：

<!–￼3–>

  • 绝对不使用标识函数（某个输入变量为 boolean, True 代表做某件事，而 False 代表做另一件事）

    • 这直接违反了一个函数只做一件事的原则
    • 将其拆分为 doSomeThingForCaseA(), doSomeThingForCaseB() 两个函数

  • 如果一个函数有两个参数，那么两者的位置就可能在使用时搞反。虽然现在的 editor 或者 IDE 可能有提示的功能，但是有时候提示确实没有那么好用。
    <!–￼4–>

在之前为了 java 考试写轮子的时候，就遇到过这个错误，当时在考场上还造成了一些小问题。
<!–￼5–>

  • 保证函数没有副作用，不做它没有声明要做的任何事情
    • 例如在 checkValidPassword() 中，不能够因为 password 被验证是 valid, 就进行 login, open new session 等操作

这一问题在我写下载网络学堂文件的脚本中也出现，但是这里无法上 code 示例因为写的真的太差已经删了【】

  • 函数要么做什么事，要么回答什么事，两者不可兼得
    • 如果函数做什么事情之后可能出错，直接 raise Exception, 而不是通过返回错误码
  • try-catch 代码块中应当只有一行代码，调用一个函数来专门处理错误的情况，这样可以保证主函数逻辑清晰。因为处理异常本身已经是一件足够复杂的事情。

直接复制黏贴 Jupyter 生成的 Markdown 文件中代码到 Hexo _posts 下面的.md文件中，渲染成的表格极其丑陋。

【啊对这里要说的是仅限于使用了pandas生成的表格会渲染丑陋，其他方便表现还不错。

捣鼓了一上午之后终于找到了比较好的解决办法。

  1. 在 Jupyter Notebook 中导出 .md 文件

  2. 利用 VSCode 中 Markdown Preview Enhanced 插件的渲染器导出成 html 文件，注意采用 “cdn host” 模式

  3. 将 html 文件放入资源文件夹

  4. 在 html 的开头加上这样三行：
    <!–￼0–>

  5. 在 /_posts/title.md 中利用 <!–￼1–> 来引入 html 文件

最终实现效果还算比较好看吧。

下面开始引用 Notebook 的技术总结【x

  1. 直接用 Jupyter Notebook 导出的 html 会使得字体变得非常丑陋，所以要导出成 .md 再用 VSCode 转换成 .html
  2. 如果不进行 (4) 的步骤，html 文件会被渲染上主站标题等等
  3. 步骤 (5) 中的长和宽可以根据 iframe 的具体大小适当调节
  4. 其他需要引用 markdown 文件等情况也可以使用这个方法，但是注意一定要在导出时使用 “cdn host” 模式，不然数学公式等无法正确显示
