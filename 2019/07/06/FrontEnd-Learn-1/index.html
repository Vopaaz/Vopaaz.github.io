之前零零散散学了一些散装的 html/css/javascript, 然后在计系原大作业里手动强撸了一个桌面应用。写的时候就感觉特别不爽，DOM 的操作语法无比繁琐，javascript 的各种事件、异步机制等等都没弄懂。

网上找了很多教程之后发现 https://www.freecodecamp.cn/ 这个网站似乎还挺靠谱的，那么趁着这个假期，以及下个学期交换可能轻松一些，好好学习一些前端知识。毕竟做出一个真正能用的应用永远是最爽的事情。

这个部分先记录一下以前学到过又遗忘的/没学到过的小点。

html

列表

  • 无序列表：unordered list, 因此标签名为 ul
  • 有序列表：ordered list, 因此标签名为 ol
  • 内含元素（每个 item）的标签名为 li.

表单

Html 表单中的 <input> 标签：

  • 是自关闭域
  • type="text" 时，是文本框，采用 placeholder 属性可以指定文本框内的占位字符。
  • required 属性（没有 ="xxx" 这样，直接写 <input type="text" required /> 即可）要求这个字段必须填写才能提交
  • type="radio" 时，是单选框
  • type="checkboxes" 时，是复选框
  • checked 属性（同样不需要 ="xxx"）可以让这个单选/复选框默认被选中

区别于以前学到的 <label> 与 <input> 标签完全分开，并且用 for 属性指定对应的 <input> 是哪个输入域（绑定的是 input 标签的 id 属性），例如：

css

css 继承

html 元素的 css 样式可以向下继承，即子元素会具有父元素的属性。

例如当对 body 设置了 color 和 font-family 等属性时，如果不进行其他设置，页面上的所有文字都会采用这种颜色和字体。

覆盖优先级

  • 根据引入方式：内联属性（例如 <h1 style="color: white;">） > 写在 .html 文件内的 style 标签内 > 写在外部文件 (css 等) 中
  • 同种引入方式，后写的覆盖先写的
  • 多个选择器指定一个元素时，会牵扯到权重计算（但是感觉基本用不到），直接记忆：id 选择器 (#id) > 类选择器 (.class) > 元素选择器 (tag)
