在 Hexo 中改变标签大小写

2019-01-15

Contents

  1. 解决方案
  2. Miscellaneous
    1. 颜色
    2. 字体
    3. 列表渲染的问题

这两天发现以前使用的博客主题 PolarBear 模板不是很完善，有些功能是缺失的，而且那个界面实在是太…素雅了🤣。后来看文档发现 PolarBear 是基于 even 写的，因此今天干脆就决定替换到 even 这个主题。

顺手发现以前写的文章 tag, 首字母有的大写有的小写看得很不爽，所以就打算全部改成首字母大写统一格式。

结果就悲剧了…在网站的 tags 页面上能够正常显示大写，但是点进每一个具体的标签就会显示 404.

解决方案

通过查看 Github 上的 Repository, 发现 tag 文件夹下的各个具体的 tag 各自有一个文件夹，标题是其名称，内有一个 index.html 页面。问题就出在这些文件夹的标题仍然是小写的，没有被同步为修改后的大写标题。查看本地的 .deploy_git 文件夹发现本地生成的 tag 文件夹下的小文件夹内容是正确的。

当时已经猜测到问题是 git 对于大小写是不敏感的，因此尝试了以下两个解决方法：

  • 删除整个 tags 文件夹，再重新 deploy
  • 把所有已经改成大写的 tags 后面加 “1”, deploy, 再删除 “1”, deploy

结果发现两个办法都无效，前者直接生成了小写文件夹名，后者在添加 “1” 后是大写，之后又变为小写。似乎是因为 git 已经保存了文件夹在小写名称时的快照，后续凡是有“忽略大小写”的重名，就会自然回到这个文件夹名称的版本。

后来开始查找资料，发现也已经有人遇到过同样的问题。

最终在这个回答下发现，其实只要调整 git 的配置就可以…

所以说到底其实是自己只用 git 客户端，而对工具本身没有了解，对命令一无所知的锅。后续要好好在这方面学习一下避免类似的情况发生。
